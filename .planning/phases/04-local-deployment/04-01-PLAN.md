# Phase 4: Local Deployment - Plan

**Phase:** 4 - Local Deployment
**Goal:** Enable easy local testing of the plugin within the user's Obsidian Vault.
**Requirements:** DEV-01

## Must-Haves
1. A script or mechanism exists to copy `main.js` and `manifest.json` into an Obsidian vault's plugin directory.

## Tasks

### [04-01-01] Add deployment script
- Update `package.json` to include a `dev` script.
- Since we don't know the exact vault path, we will create a `copy-to-vault.mjs` script that uses a `.env` file containing `VAULT_PATH`.
- Install `dotenv` as a dev dependency.
- Create an `.env.example` file with `VAULT_PATH=`.
- Add `copy-to-vault.mjs` which reads the vault path and copies `main.js` and `manifest.json` to `${VAULT_PATH}/.obsidian/plugins/obFileType/`.

### [04-01-02] Update esbuild to auto-copy
- Alternatively, we can just modify `esbuild.config.mjs` to optionally copy files to `VAULT_PATH` if provided, which is a cleaner approach often used in Obsidian plugin development. We'll update `esbuild.config.mjs` to copy the files to the plugin directory in the vault after building.
