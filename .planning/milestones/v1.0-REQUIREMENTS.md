# Requirements: obFileType

**Defined:** 2026-06-30
**Core Value:** Enable users to seamlessly edit arbitrary text-based files directly within Obsidian without relying on external editors.

## v1 Requirements

### File Handling

- [ ] **FILE-01**: User can open a text-based file (e.g., .json, .yaml, .c) inside Obsidian.
- [ ] **FILE-02**: User can view the contents of the opened text file.
- [ ] **FILE-03**: User can edit the contents of the opened text file.
- [ ] **FILE-04**: User can save the edited text file back to the disk.

### Editor Features

- [ ] **EDIT-01**: User can see syntax highlighting for common text formats.

### Settings

- [ ] **SETT-01**: User can configure a list of file extensions that the plugin should treat as text.
- [ ] **SETT-02**: User can enable or disable the plugin from Obsidian's Community Plugins page.

## v2 Requirements

### Advanced Editing

- **EDIT-02**: User can configure word wrap and line numbers.
- **EDIT-03**: User can use find and replace within the file.

## Out of Scope

| Feature | Reason |
|---------|--------|
| IDE features (debugging, intellisense, etc.) | Plugin is meant to be a simple text viewer/editor |
| Binary file support (e.g., .xlsx, .pptx, images) | Obsidian does not handle arbitrary binary rendering in a text editor context, out of scope for this plugin |
